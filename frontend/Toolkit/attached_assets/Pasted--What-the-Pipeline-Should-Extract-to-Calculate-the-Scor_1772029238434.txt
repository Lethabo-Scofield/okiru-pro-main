#### Pipeline Extraction Overview
- **Input**: Filled Excel (main file) + optional PDFs (e.g., cert for cross-validation).
- **Extraction Strategy**:
  - Sheet-by-sheet parsing (use openpyxl or similar; fuzzy match names like "Management Control" or "Procurement").
  - Row-by-row for lists (ignore blanks; stop at empty rows or totals).
  - Column mapping: Fuzzy headers (e.g., "Supplier Name *" → "supplierName"; handle asterisks as required flags).
  - Transformations: Convert text % to decimals (e.g., "100.00%" → 1.0), dates to ISO (e.g., "28 February 2025" → "2025-02-28"), sum aggregates (e.g., total spend).
  - Validation: Flag errors (e.g., missing Race in MC → "Incomplete – default 0 pts"), warnings (e.g., negative spend → "Review").
- **Output JSON Structure** (for scorecard & docs):
  ```json
  {
    "client": { /* From Information Request / Finance */ },
    "financials": { /* Revenue, NPAT, TMPS, etc. */ },
    "ownership": [ /* Array of shareholders */ ],
    "mc": [ /* Array of employees */ ],
    "skills": [ /* Array of trainings */ ],
    "procurement": [ /* Array of suppliers */ ],
    "esd": [ /* Array of contributions */ ],
    "sed": [ /* Array of contributions */ ],
    "yes": [ /* Array of youth employees */ ],
    "calculated": { "pillars": { "ownershipPts": 25, ... }, "totalPts": 102, "level": 1, "gaps": {...} },
    "errors": [ "Missing ID in MC row 3" ]
  }
  ```
- **Calcs Enabled**: With this extracted data, run formulas (e.g., Ownership: sum BO% / target * pts; Procurement: spend * recognition % / TMPS * pts).
- **Outputs Generated**: Dynamically fill templates (e.g., PDF with jsPDF, PPTX with pptxgenjs, Excel with exceljs) using the JSON.

#### Sheet-by-Sheet Extraction Needs (From Filled Template)

1. **Information Request Sheet** (General Client Details – Extract First for Context)

     - Entity Name: From row1 (e.g., "Thandanani Transport")
     - Financial Info: Revenue, NPAT, Payroll (historical & forecasts; e.g., rows 9-15: "Revenue" → value in next cell)
