# Configuration & Secrets Reference

This is the single source of truth for every environment variable used
across the data layer packages. Keep it up to date when adding new providers.

---

## How environment variables work here

Every provider package reads configuration from **environment variables** at
startup — never from hardcoded values or committed files. The pattern is:

```
.env.example   ← committed to git    (dummy values, safe to share)
.env           ← NOT committed        (real values, lives on your machine / CI / vault)
```

To get started locally:

```bash
# 1. Copy the example file
cp .env.example .env

# 2. Fill in your real values
# (open .env in your editor)

# 3. Load it before running (PowerShell)
Get-Content .env | ForEach-Object {
  if ($_ -match '^\s*([^#][^=]+)=(.*)$') {
    [System.Environment]::SetEnvironmentVariable($matches[1].Trim(), $matches[2].Trim())
  }
}
```

## Master variable table

| Variable | Used by | Required | Example value | Description |
|---|---|---|---|---|
| `DATA_PROVIDER` | All apps | Yes | `postgres` | Which provider the app uses. Must match a key registered in the provider registry. Allowed values: `postgres`, `mongodb` |
| `DATABASE_URL` | Postgres providers | Yes (when `DATA_PROVIDER=postgres`) | `postgresql+asyncpg://user:pass@localhost:5432/mydb` | Full Postgres connection string. TypeScript uses `postgresql://`, Python uses `postgresql+asyncpg://` |
| `MONGO_URL` | MongoDB providers | Yes (when `DATA_PROVIDER=mongodb`) | `mongodb://localhost:27017` | MongoDB connection string. Does NOT include the database name — that is set separately via `MONGO_DATABASE` |
| `MONGO_DATABASE` | MongoDB providers | Yes (when `DATA_PROVIDER=mongodb`) | `my_app` | The MongoDB database name to use |
| `DB_POOL_SIZE` | Postgres providers | No | `10` | Max number of connections to keep open in the pool. Defaults to 10 if not set |

## Connection string formats

### PostgreSQL — TypeScript (`typescript-postgres`)

```
postgresql://[user]:[password]@[host]:[port]/[database]
```

Example:
```
DATABASE_URL=postgresql://postgres:secret@localhost:5432/my_app
```

### PostgreSQL — Python (`python-postgres`)

The Python provider uses SQLAlchemy with asyncpg. The connection string
**must** include the `+asyncpg` driver suffix:

```
postgresql+asyncpg://[user]:[password]@[host]:[port]/[database]
```

Example:
```
DATABASE_URL=postgresql+asyncpg://postgres:secret@localhost:5432/my_app
```

> Why the difference? SQLAlchemy uses the prefix to pick which driver to
> load. Without `+asyncpg` it falls back to the synchronous psycopg2 driver
> which does not work with async/await.

### MongoDB — TypeScript & Python

```
mongodb://[user]:[password]@[host]:[port]
mongodb+srv://[user]:[password]@[cluster].mongodb.net   ← Atlas / cloud
```

Example:
```
MONGO_URL=mongodb://localhost:27017
MONGO_DATABASE=my_app
```

## Pool size guidance

| Environment | Recommended `DB_POOL_SIZE` | Reason |
|---|---|---|
| Local dev | `3–5` | You're the only user |
| Staging | `5–10` | A few concurrent test runs |
| Production | `10–20` | Tune based on load and DB connection limits |

Keep in mind: if you run 3 app instances with `DB_POOL_SIZE=10`, your database
will see up to 30 simultaneous connections. Stay within your database plan's
connection limit.

## Secrets in production

Never store real credentials in `.env` files that are deployed to servers.
Use a secrets manager instead:

| Platform | Tool |
|---|---|
| Kubernetes | Kubernetes Secrets + inject as env vars via `envFrom` |
| AWS | AWS Secrets Manager or Parameter Store |
| Azure | Azure Key Vault |
| GCP | Google Secret Manager |
| Self-hosted | HashiCorp Vault |

The application code stays the same — it always reads from environment
variables. Only the mechanism that *populates* those variables changes.

## CI / CD

In your CI pipeline (GitHub Actions, GitLab CI, etc.):

- Store secrets in the platform's **encrypted secret store**, not in the repo.
- Inject them as environment variables in the job definition.
- Never print them in logs (`echo $DATABASE_URL` will expose the password).

Example (GitHub Actions):

```yaml
env:
  DATABASE_URL: ${{ secrets.DATABASE_URL }}
  DATA_PROVIDER: postgres
```

## What must never be committed

| File / pattern | Why |
|---|---|
| `.env` | Contains real passwords |
| `.env.local` | Same |
| `.env.production` | Same |
| `.env.development` | May contain real local credentials |
| Any file with a real password, API key, or connection string | Obviously |

These are all covered by the root `.gitignore`. If you accidentally commit
one, rotate the credential immediately — git history is public even if you
delete the file in a later commit.
