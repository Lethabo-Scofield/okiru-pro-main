Prompt for Replit

You have full access to the Okiru project. Implement SEO for the B-BBEE Certificate Hub so that individual certificates rank on Google.

Objective

Make every certificate indexable, searchable, and SEO-optimized so that queries like:

“B-BBEE certificate {company name}”
“{company name} BEE status”
“Level 1 BEE companies South Africa”

return Okiru pages.

1. Create Dynamic Certificate Pages (CRITICAL)

Implement a route:

/certificates/[slug]

Slug format:

{company-name}-{certificate-number}

Example:

/certificates/absa-123456789

Fetch certificate data from API using slug or ID.

2. Server-Side Rendering (IMPORTANT)

Ensure certificate pages are rendered using SSR or SSG (NOT client-only rendering).

If Next.js → use getServerSideProps or generateStaticParams
If Express → render HTML on server
3. SEO Meta Tags (per page)

Dynamically generate:

Title

B-BBEE Certificate – {Company Name} | Level {Level} | Okiru

Meta Description

View and download the B-BBEE certificate for {Company Name}. Level {Level}, issued by {Verification Agency}, valid until {Expiry Date}.
4. Page Content (VISIBLE HTML)

Render all certificate fields in clean HTML:

Company Name (H1)
B-BBEE Level
B-BBEE Score
Black Ownership %
Black Women Ownership %
Verification Agency
Certificate Number
Expiry Date
Download button (PDF)

Ensure content is not hidden behind JS.

5. Add Structured Data (JSON-LD)

Inject this into <head>:

{
  "@context": "https://schema.org",
  "@type": "DigitalDocument",
  "name": "B-BBEE Certificate - {Company Name}",
  "creator": "{Verification Agency}",
  "dateValid": "{Expiry Date}",
  "identifier": "{Certificate Number}",
  "about": {
    "@type": "Organization",
    "name": "{Company Name}"
  }
}
6. Create SEO Landing Pages

Add static pages:

/certificates/level-1
/certificates/level-2
/certificates/black-owned

Each page must:

Have 100–200 words of descriptive text
List certificates (server-rendered)
Link to individual certificate pages
7. Sitemap Generation

Create:

/sitemap.xml

Include:

All certificate pages
All category pages

Auto-update daily.

8. Internal Linking
From homepage → link to certificate hub
From lists → link to certificate pages
From certificate page → link to related certificates
9. Performance Requirements
Page load < 2 seconds
Avoid blocking scripts
Optimize images and payload size
10. Validation Checklist

Ensure:

Page loads without JS
Page is crawlable (view page source shows full content)
Meta tags are correct
JSON-LD is valid
URLs are clean and readable
Deliverables
Working /certificates/[slug] pages
SEO meta + structured data implemented
Sitemap generated
At least 1 working category page
Example certificate fully optimized

Do not overcomplicate. Focus on:

Indexable pages
Clean content
Correct metadata
