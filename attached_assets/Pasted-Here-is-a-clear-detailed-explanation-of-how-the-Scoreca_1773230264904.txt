Here is a clear, detailed explanation of **how the Scorecard should look and what it MUST include**, directly based on the structure, logic, and content of the Excel document ("Okiru Toolkit (RCOGP)_Template_v.1.0.xlsx").

This mirrors the Excel's **Summary Scorecard** sheet (main overview), **pillar-specific scorecards** (Ownership, MC, Skills, etc.), and how data flows between sheets (Financials → targets → calculations → display).

### 1. Overall Scorecard Philosophy (from Excel)
- **One main view** = Summary Scorecard (like Sheet 4)
- **Expandable rows** for each pillar → shows full sub-indicators (like how Ownership expands to Voting Rights, Net Value, etc.)
- **Real-time calculations** pulled from input sheets (Financials, EAP, employee data, procurement spend, etc.)
- **Priority elements highlighted** with sub-minimum warnings (40% rule → discount if failed)
- **Totals & recognition level** always visible at top/bottom
- **Gaps, % achieved, projected values** shown clearly
- **Toggle options**: Wrap text, full precision (2 vs 4 decimals), expand/collapse all

### 2. Top Section – Hero / Status Bar (Always Visible)
This should be fixed/sticky at the top (like Excel row 1–3 style summary).

Must include:
- **Company Name & Period**  
  Example: "Lakeside Trading – 1 Mar 2025 – 28 Feb 2026 (Current YTD)"
- **Current Level & Recognition %**  
  Example: "Level 6 Contributor – 60% Recognition"  
  Color-coded: Green (1–3), Blue/Teal (4), Amber (5–6), Red (7–Non-compliant)
- **Total Points**  
  Example: "68.4 / 120 pts" + progress bar (68.4 / 120 = 57%)
- **Discount Status** (red banner if active)  
  Example: "Level discounted by 1 – Ownership Net Value sub-minimum failed (2.8 / 8 pts)"
- **Projected Level** (if no more action)  
  Example: "Projected: Level 7 (no further spend)"
- **YES Status**  
  Example: "YES Tier 1 – +1 level possible (8/12 youth placed, 2.5% absorption)"

### 3. Main Scorecard Table (Core Content)
Table format like Excel Summary Scorecard, with expandable rows.

**Columns** (fixed widths for readability):
1. Element / Pillar (left-aligned, clickable to expand)
2. Weight / Target Points
3. Prior Year
4. Actual YTD
5. Gap (red if negative)
6. Projected (end-of-year estimate)
7. % Achieved (progress mini-bar + number)
8. Status (icons: ✓ On Track / ⚠ At Risk / ✗ Critical + text)
9. Priority? (Yes/No or red star)
10. Sub-minimum (Passed/Failed/N/A – only for priority pillars)

**Rows** (in exact Excel order):
- Ownership (25 pts) → expands to 7 sub-lines + Net Value
- Management Control (19 pts) → expands to 13 sub-lines (board → disabilities)
- Skills Development (25 pts) → expands to 5 sub-lines (learning 6, bursaries 4, disabled 4, learnerships 6, absorption 5)
- Preferential Procurement (29 pts) → expands to 6 sub-lines + 2 bonuses (graduation/jobs)
- Supplier Development (10 pts) → expands to 1 main + bonuses if applicable
- Enterprise Development (7 pts) → expands to 3 sub-lines + 2 bonuses
- Socio-Economic Development (5 pts) → expands to 1 line
- **Grand Total** (120 pts) – bold row at bottom
- **YES Bonus** – separate row or highlighted tile

**Expandable behavior** (like Excel grouped rows):
- Click row → shows sub-indicators indented below
- Sub-rows show: Name, Weight, Target, Actual, Gap, % Achieved, Formula tooltip

### 4. Priority Elements Section (Must Be Very Visible)
Separate card or red/amber highlighted box below hero.

Must show:
- Ownership Net Value → X / 8 pts → Failed if < 3.2
- Skills Development → X / 25 pts → Failed if < 10
- Preferential Procurement → X / 29 pts → Failed if < 11.6
- Supplier Development → X / 10 pts → Failed if < 4
- Enterprise Development → X / 5 pts → Failed if < 2

Warning text:
"Level will be discounted by 1 until at least 3 priority sub-minimums are met."

**Demographic Breakdowns** (EAP influence)
- Skills → under "Learning Programmes for Black People": pie chart / table showing % split (African male, African female, Coloured male, etc.) based on selected EAP region

**Bonus Indicators**
- Preferential Procurement → +1 graduation, +1 jobs
