## [0.1.0] — 2026-04-29

Initial release of all six packages.

### Added

**Core**
- `@company/data-layer` / `company-data-layer` — shared interfaces: `IUnitOfWork`, `IRepository`, `IDataAccessFactory`, `IProviderRegistry`
- `@company/data-layer/testing` subpath — `FakeUnitOfWork` and `FakeDataAccessFactory` for unit tests without a real database

**PostgreSQL provider**
- `@company/data-layer-postgres` / `company-data-layer-postgres` — `PostgresUnitOfWork`, `PostgresDataAccessFactory`, `PostgresUserRepository`
- Drizzle ORM (TypeScript) and SQLAlchemy 2 async (Python)
- Migration support via `drizzle-kit` (TypeScript)
- `loadPostgresConfig()` — reads `DATABASE_URL` and `DB_POOL_SIZE` from env, fails fast if missing

**MongoDB provider**
- `@company/data-layer-mongo` / `company-data-layer-mongo` — `MongoUnitOfWork`, `MongoDataAccessFactory`, `MongoUserRepository`
- MongoDB Node.js driver (TypeScript) and motor 3 (Python)
- `loadMongoConfig()` — reads `MONGO_URL` and `MONGO_DATABASE` from env, fails fast if missing

**Observability**
- OpenTelemetry spans on `commit()` and `rollback()` in all four UoW implementations
- `db.provider` span attribute tags each span with `"postgres"` or `"mongodb"`
- Correlation ID (trace ID) logged to stderr on commit/rollback failure

**Example apps**
- `example-express` — Express app showing request-scoped UoW via middleware, `DATA_PROVIDER` switching via `.env`
- `example-fastapi` — FastAPI app showing request-scoped UoW via `Depends` + `yield`, `DATA_PROVIDER` switching via `.env`

**Documentation**
- `CONFIG.md` — all environment variables, connection string formats, pool sizing, production secrets guidance
- `MIGRATION-PLAYBOOK.md` — step-by-step guide for migrating existing services to the data layer
- `PUBLISHING.md` — semver rules, release checklist, internal registry setup (this file)
- `.env.example` in every provider and example package
