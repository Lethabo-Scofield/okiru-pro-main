We need to improve the onboarding and team invitation system for the existing platform.

Current issue:
Team invitation emails are being sent from the platform email (Okiru email). This should not behave like a platform-generated registration email.

Required behavior:
The invite flow should behave similarly to Google Drive sharing.

Example:

* User creates or manages a compliance project
* User invites team members
* Invited member receives an invite that appears as if it was initiated by that specific user/project owner
* The system should clearly indicate who invited them and which project they are joining

Key requirements:

1. Invitation System

* Refactor the invitation email logic
* Do not make invites feel like generic Okiru system emails
* Invitation context must include:

  * inviter name
  * inviter company/team
  * project name
  * role being assigned
* Improve email UX and wording to feel collaborative

2. Onboarding Reliability

* Ensure all onboarding data is saved correctly
* Validate:

  * user creation
  * role assignment
  * organization/project linking
  * invite acceptance state
  * token expiry
  * onboarding completion state
* Prevent duplicate onboarding records
* Add proper error handling and logging

3. Multi-Tenant Isolation
   Critical:
   No client logic, onboarding state, permissions, or project data should ever leak across tenants.

Requirements:

* Strict tenant isolation
* Every query and API route must validate tenant/project ownership
* No shared onboarding state between clients
* No shared invitation context between organizations
* Certificates are the only allowed shared/global resource

4. Security

* Secure invite tokens
* Expiring invite links
* Validate invited email before accepting invite
* Prevent privilege escalation
* Prevent unauthorized project access
* Ensure session protection and access control are enforced across onboarding flows

5. Audit + Testing

* Add tests for:

  * invite creation
  * invite acceptance
  * expired invites
  * duplicate invites
  * tenant isolation
  * onboarding persistence
* Add logs for onboarding failures and invite events

Goal:
Make onboarding production-ready, secure, collaborative, and enterprise-grade.
