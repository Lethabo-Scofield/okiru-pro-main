Implement a Company Onboarding flow as part of user signup.

Goal:
Capture structured company data immediately after account creation and link it to future certificate uploads.

Flow:
- Step 1: User signs up (auth system already exists)
- Step 2: Redirect to onboarding form
- Step 3: Save data and attach to user profile
- Step 4: Redirect user to /certificates (and open upload if they came from there)

Database:
Create a table: company_profiles

Fields:
- id
- user_id (foreign key)
- company_name (required)
- role
- bee_level
- employee_range
- industry
- annual_revenue
- acquisition_source
- tools_used (array)
- biggest_challenge (text)
- created_at
- updated_at

Requirements:
- Validate required fields
- Allow “Other” to capture custom input
- Store tools_used as array
- Link onboarding to user_id
- Ensure onboarding runs once per user (unless editing later)
- Add API endpoint:
  POST /onboarding
- Add GET /onboarding/me

UX:
- Clean, simple form
- Progress indicator (Step 2 of 2)
- Clear CTA: “Continue to Certificates”

Definition of done:

- Data is saved and linked to user
- System can query company data later (for analytics or linking certificates)
